title：Diffraction Imaging

We all know that light has two character: wave and particle. The principle phenomenon for wave is the diffraction. How about using light to imaging a specific pattern?

— Well, we know that if we want to use sunlight to imaging pictures, just shadow specific part of the light beam.


However, for diffraction, the light can not be block by a simple barrier : light will pass a object which is as small as its wavelength. So, how to using diffraction to imaging is an unimaginable problem.


Thanks to the growing science and technology, we can compute this case and reappear a diffraction imaging in lab.

What we need is just

  1. a lase beam
  2. LC optical shutter

Physics Image

We can easily image light beam consists of a great number of light line. It is fine in this situation and will not lose much physics nature.

Firstly, diffraction required that every light line will split into hundreds sub-line after a small hole or barrier.


These amount lines are objected a very complex physics rule, but approximately, we can regard this rule as
$$
f(x-x_o,y-y_0,z,a)=\frac{4 a^2}{z}e^{\frac{i \pi}{z} ((x-x_0)^2+(y-y_0)^2)}sinc(\frac{2\pi a}{z}(x-x_0))sinc(\frac{2\pi a}{z}(y-y_0))
$$
where value $f$ is the wave function in space. And $|f|^2$ is just the intensity of this point.

where the $sinc$ function is defined by $y=\frac{sin(\pi x)}{(\pi x)}$

if you want to know how this function comes, please read Fraunhofer diffraction

and Collins formula in Matrix Optical.

The above formula suggests the distribution of diffracted plan wave light. And it is actually a 3D image

So, if we want to know the wave function on the output surface, we need realize that it is the sum of all the light from input surface.

Now, we can easily write the output wave function.
$$
Output=\int_{x_0} \int_{y_0} f(x-x_0,y-y_0)\times E(x_0,y_0) dx_0dy_0
$$
Here, $E(x_0,y_0)$ is the original wave function at input surface $(x_0,y_0)$.

we can immediately realize this formula is actually a convolution. And now we can using Fourier Transform to simple the integral
$$
\mathfrak{F}(Output)=\mathfrak{F}(field)\mathfrak{F}(E)
$$
this is also one idea to simplify computation.

Back to our purpose: we want to use diffraction to imaging what we want to get.

For example, we want to get a picture like a rabbit.


And due to the limitation about eyes and information. We suppose that our human can only distinguish the light one and dark one, i.e. we can only receive intensity information ( this is totally wrong, because our eyes can actually analyze phase information which is helpful for our 3D feeling)

so, what we really care about is not the wave function $Output$, otherwise, the intensity of output $|Output|^2$.

As we discuss above, for diffraction, we can not get the inverse picture that when light pass through it, it will form a rabbit. What we can only do is to find a gesture to make our final picture get similar to rabbit.

Let me summarize. For a ordinary light imaging problem, we can easily get a gesture, put it under the sunlight, we can receive a wolf, but for a diffraction problem, we can not get that according gesture, we can not get a wolf, but the similar thing as it, maybe a dog.

Math Nature

Actually, if we abstract the main principle of physics image, we can get a very simple math question.

How to minimalize the difference between what we compute and what we want:
$$
\min ||F(input)-taget||
$$
where the $||.||$ is the Weighted Frobenius Norm

  • the $input$ is our gesture, it is a matrix

  • the $target$ is our wolf, it is a matrix

  • Because it is a gesture, we can only control the shadow and pass. (Actually, if we have a good device, we can also control the phase and intensity, then there is not the same problem )

  • Because it is a wolf, so we only compute the intensity

  • so the $input$ is a matrix contain only $0$ and $1$ which mean close and open respectly.
  • so the $target$ is a matrix contain only non-negative element due to it is square module.

#Method

If the gesture contain can be any matrix, this problem is a typical multi-variable optimization problem. Using LBFGS is a good choose. However, now, the $input$ matrix is only the open-close matrix which map to a discrete optimization problem.

In my opinion, to solve this problem, using Monte Carlo simulation is a good way. Adding anneal element, this is a good way.
